Node.js 로 서버를 만들어 쓰는 이유 ( SNS나 채팅서비스에 많이 사용 )


일반 서버는 한번의 1개의 요청을 처리할수 있다. 


Node.js 는 모든 요청을 한번에 받는다

그리고 모든 요청중 순서에 상관없이 처리 속도가 빠른 것부터 결과를 가져다줌 (  Non-blocking I/O )
채팅서비스나 SNS에 많이 쓰는이유는 요청이 매우 많은 부분에서 처리하기 좋음 
( 단 일반 서버에서도 스케일리(뭔지모름) 을 하거나 멀티쓰레드를 사용하면 사용할 수 있긴함. )
코드가 매우 짧고 쉬워서 빠른 개발 가능 ( 많은 요청, 작은 데이터 를 주고받기 적합함 )


Node.js 의 단점은

모든 요청이 하나의 스레드에서 처리되기 때문에 CPU 연산 즉 많은 처리량을 요구하면
당연하게도 스레드 하나가 감당하기 힘드므로 비디오 처리, 대규모 데이터 처리 같은
대규모 데이터 처리 서버로는 권장하지 않는다.
